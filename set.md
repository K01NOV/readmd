# Практическая работа №3: Группировка тестовых наборов

## 1. Группировка по функциональным модулям
**Цель:** Проверка работоспособности конкретных бизнес-вертикалей. Это основной вид группировки для проведения регрессионного тестирования при изменении кода в конкретном модуле.

| Название набора | Описание | Характеристика | ID тест-кейсов |
| :--- | :--- | :--- | :--- |
| **Identity & Access** | Проверка всех механизмов входа, регистрации и прав доступа. | По функциональному модулю (Аутентификация + Права) | TC001, TC002, TC003, TC021, TC027, TC035 |
| **E-commerce Core** | Тестирование основного пути пользователя: каталог, поиск и корзина. | По функциональному модулю (Каталог) | TC010, TC011, TC012, TC013, TC014 |
| **Order Management** | Полный цикл работы с заказами: от создания до возврата и истории. | По функциональному модулю (Заказы) | TC007, TC008, TC009, TC018 |
| **User Profile & Data** | Набор для проверки личного кабинета и данных пользователя. | По функциональному модулю (Пользователи) | TC004, TC005, TC006, TC022, TC023, TC025, TC032 |

## 2. Группировка по приоритету
**Цель:** Позволяет эффективно распределять ресурсы в условиях ограниченного времени. В первую очередь всегда выполняется "Smoke" набор.

| Название набора | Описание | Характеристика | ID тест-кейсов |
| :--- | :--- | :--- | :--- |
| **Smoke Test (High)** | Критически важные тесты, проверяющие основную жизнеспособность продукта. | По приоритету (Высокий) | TC001, TC002, TC004, TC007, TC010, TC015, TC016, TC021, TC027, TC031, TC033, TC035 |
| **Medium Priority Suite** | Проверка стандартных функций, которые важны, но не блокируют основной бизнес-процесс. | По приоритету (Средний) | TC003, TC005, TC006, TC008, TC011, TC012, TC014, TC017, TC018, TC022, TC023, TC024, TC028, TC029, TC030, TC032, TC034 |
| **Low Priority / Backlog** | Проверки второстепенных функций и системных процессов. | По приоритету (Низкий) | TC009, TC013, TC019, TC020, TC025, TC026 |

---

## 3. Группировка по виду тестирования
**Цель:** Разделение кейсов по их технической направленности. Помогает определить, какие тесты можно автоматизировать, а какие требуют ручного или специализированного подхода.

| Название набора | Описание | Характеристика | ID тест-кейсов |
| :--- | :--- | :--- | :--- |
| **Security & Stability** | Тестирование безопасности данных и устойчивости системы к внешним сбоям. | По виду (Безопасность и надежность) | TC021, TC027, TC031, TC033, TC035 |
| **Non-Functional & Sys** | Проверки производительности API, логирования и обновлений. | По виду (Нефункциональное тестирование) | TC019, TC020, TC028 |
| **UI & Localization** | Проверка внешнего вида приложения и корректности перевода. | По виду (Тестирование интерфейса) | TC014, TC034 |

---

## 4. Группировка по бизнес-процессам (End-to-End)
**Цель:** Проверка сквозных сценариев, имитирующих реальный путь клиента от точки А до точки Б.

| Название набора | Описание | Характеристика | ID тест-кейсов |
| :--- | :--- | :--- | :--- |
| **Customer Purchase Way** | Сценарий покупки: поиск товара, добавление в корзину, купон и оплата. | По бизнес-процессу (Покупка) | TC012, TC010, TC015, TC016, TC007 |
| **Communication Flow** | Проверка всех видов оповещений пользователя о событиях в системе. | По бизнес-процессу (Уведомления) | TC022, TC029, TC030 |
